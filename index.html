Part 2: HTML Tags and the Basics of the Web

The Basics of the Web

There are 5 basic components to understand about the web:
The World Wide Web (WWW) - (1) HTML (Hypertext Markup Language) or hyperlinks that connect the web; invented in the early 1990's.
Major pieces of the web also include: (2) You, (3) your computer/browser, (4) the internet, (5) and servers.

HTML - Hypertext Markup Language

HTML (or Hypertext Markup Language) is made up of:
(1) Text Content = what you see
(2) Mark Up = what it looks like
(3) References to other documents (e.g videos and images like this one - )
(4) Links to other pages

There are also a variety of attributes that can be added to HTML to further enchance the web page.

The "a href" tag is used to link other webpages to yours. It can be used to link sites like Reddit or CNN.

Images can also be added using the "img" tag.

Note: the use of img tags do not require a closing tag.

You can also use Google as a resource to add links in HTML since it's impossible to remember everything!

Whitespace, Inline, and Block Elements

The use and understanding of whitespace is important - it allows you to place text on multiple lines.

Two types of elements to understand are Inline and Block elements:
(1) Inline - Examples of Inline elements include the use of the following tags: b, em, img, br, and span
(2) Block - Examples of Block elements include the use of the following tags: p and div
**Inline tags behave differently from Block tags; Block elements can add height and width to a web page.

Tags
- An HTML tag is contained within angled brackets
- Most tags have an opening and closing tag
- Some tags are void or do not require a closing tag - i.e. the "br" tag

Element - an HTML element refers to everything within a set of opening and closing tags.

Attribute - this is a property of an HTML element (i.e. - to set the HREF attribute of an anchor tag to the Udacity URL you would write a href="http://www.udacity.com")

Part 3: Adding Style to Structure with CSS

"The Tree-Like Structure of a Page (DOM)"

1. HTML and CSS are both "languages"
2. HTML controls the "structure" of the web
3. CSS controls the "style" of the page (how it looks)
4. When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page

CSS

CSS or Cascading Style Sheets is code written to control the "style" of HTML elements. There are three ways to write CSS code into files:
1. Written into the HTML head code itself (not good)
2. Link HTML to a separate CSS file (good)
3. Write your style inline with your HTML (not good)

Box Sizing

Four important things to consider when box sizing CSS:
1. HTML elements are boxes and each box has 4 components
2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
3. 2 techniques you can use to deal with sizing issues:
(i) Set the sizes in terms of percentages rather than pixels
(ii) Set the box-sizing attribute to border-box for every element

** Note - Different browsers work slightly differently. Sometimes this cuases different browsers to display code differently.

Box Repositioning

1. DIVs are block elements (as opposed to inline) so by default they take up the entire width of a page.
2. Adding particular rules to the appropriate CSS will override this behavior and let DIVs appear next to each other.
